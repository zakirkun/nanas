# Security Policy

If you discover a security vulnerability in Nanas, please do not publish exploitable details in a public issue.

Use a private maintainer contact for the repository or open a minimal public issue asking for a secure reporting channel. Include enough context for maintainers to triage the report, but avoid posting secrets, exploit payloads, or customer data.

Maintainers should acknowledge serious reports in a reasonable timeframe and coordinate fixes before public disclosure.
